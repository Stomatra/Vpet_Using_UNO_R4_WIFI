# Vpet_Using_UNO_R4_WIFI

主要目的：在UNO R4 WIFI上面设计一个虚拟桌宠（用到了SD卡，闪存不够用，5张图片全是在8G内存卡里面，已经格式化为符合要求的FAT32格式），主要功能如下：

1. 在正常状态，无操作的时候，显示图片img0
2. 在触碰屏幕的时候，读取img1,img2,img3中的随机一个图片，然后过几秒回到初始状态
3. 在触碰屏幕5次的时候，强制把鼠标拖拽到右下角，并读取img4，相当于咬住鼠标了，然后过几秒回到初始状态

已经实现的功能：

1. 触摸5次之后控制鼠标
2. 图片的转换逻辑已经实现
3. 基本大体的框架已经完成，接下来就是改各种各样层出不穷的bug

目前出现的一大堆问题：

1. 首先是图片无法正常读取的问题，会出现各种条纹状的色块，或者杂七杂八的色块而不是完整图片，并且一直提示“Cannot open the pic file”
2. 然后是错误的触屏探测的问题，从初始化之后一直都是-4096,-4096的位置不断进行错误探测，即使手并没有碰到屏幕
